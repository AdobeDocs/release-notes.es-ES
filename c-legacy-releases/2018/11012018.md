**Correcciones, mejoras y desaprobaciones**

Hemos actualizado el nombre del informe Historial saliente a Historial de archivos salientes. El nombre anterior llevaba a algunos clientes a pensar que el informe muestra los datos salientes para los destinos HTTP. En realidad, el informe cubre los archivos entregados a ubicaciones S3 o FTP.

**Problemas conocidos**

* La versión más reciente de Safari incluye las herramientas Intelligent Tracking Prevention (ITP) 2.0. Se ven afectadas las métricas Audiencia direccionable para los usuarios de Safari y la recopilación de datos mediante la señal h_referer. Lea el tema sobre el tráfico de Safari como [causa de tasas de coincidencia bajas para audiencias direccionables ](https://marketing.adobe.com/resources/help/es_ES/aam/addressable-audiences-match-rates.html) y [recopilación de datos con h_ prefix](https://marketing.adobe.com/resources/help/es_ES/aam/r_tb_variable_prefixes.html).
* La versión de [Exclusiones de características en el modelado algorítmico](https://marketing.adobe.com/resources/help/es_ES/aam/trait-exclusion-algo-models.html) introdujo un problema para clientes que utilizan [Controles de acceso basados en roles](https://marketing.adobe.com/resources/help/es_ES/aam/c_administration.html) . Al crear un modelo nuevo, si solo selecciona fuentes de datos a las que tenga acceso, podrá ver las características correspondientes en la ventana **Exclusiones**. Sin embargo, si selecciona alguna fuente de datos a la que no tenga acceso (además de aquellas a las que sí lo tiene), se mostrará una lista en blanco. Anule la selección de las fuentes de datos a las que no tiene acceso para poder ver las características. (AAM-42380)
